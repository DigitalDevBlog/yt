<objective>
## Goal
Add a `-s`/`--summarize` mode that fetches the transcript, pipes it to
`claude -p "<prompt>"`, and writes the returned summary to a sanitized
`<title>.md` file in the current directory â€” at zero API cost. The prompt
defaults to `YT_SUMMARY_PROMPT` from `~/.config/yt/.env` (with a built-in
fallback) and can be overridden per-run with `-p`/`--prompt`.

## Purpose
This is the headline extension: turn `yt` from a fetch tool into a
transcript-to-summary pipeline for LLM workflows, using the subscription-backed
`claude` CLI rather than the paid API.

## Output
Modified `src/main.rs` and `Cargo.toml` (adds `indicatif`).
</objective>

<acceptance_criteria>

## AC-1: Summarize writes a titled markdown file
```gherkin
Given a valid YouTube URL, a configured YOUTUBE_API_KEY, and `claude` on PATH
When `yt --summarize <url>` runs
Then the transcript is sent to `claude -p` and the returned summary is written
     to "<sanitized-title>.md" in the current directory
And nothing is written to stdout (the file path is reported on stderr)
```

## AC-2: Prompt resolution and override
```gherkin
Given YT_SUMMARY_PROMPT is set in ~/.config/yt/.env
When `yt --summarize <url>` runs without --prompt
Then that env value is used as the claude prompt
And when `-p "<custom>"` is supplied, the custom prompt is used instead
And when neither is set, a built-in default prompt is used
```

## AC-3: Filename sanitization
```gherkin
Given a video whose title contains filesystem-illegal characters (/ \ : * ? " < > |)
When the output filename is derived
Then those characters are removed/replaced, whitespace is collapsed, the name is
     trimmed and length-capped, and ".md" is appended
And when the sanitized title is empty, the video id is used as the filename
```

## AC-4: Missing `claude` binary is handled
```gherkin
Given the `claude` binary is not on PATH
When `yt --summarize <url>` runs
Then the tool exits with a clear error explaining claude is required
And does not write a partial or empty .md file
```

## AC-5: Stepped progress display (summarize flow)
```gherkin
Given `yt --summarize <url>` runs in a terminal
When the pipeline executes
Then progress is shown on stderr as a vertical list of steps â€” "Fetching video
     details", "Downloading transcript", "Summarizing with claude" â€” each on its
     own line with a "â—¦" bullet and a "|" connector between steps
And the active step shows a spinner, completed steps show "âś“", a failed step "âś—"
```
