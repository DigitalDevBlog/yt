<objective>
## Goal
Capture the YouTube video title from the Data API and surface it in output: add
it to the default JSON object and expose a `--title` flag for title-only output.
Fetch it by extending the existing `videos` call to also request `snippet`, so
no additional HTTP request is made.

## Purpose
The title is required by the Phase 2 `--summarize` feature (to name the
`<title>.md` output file) and is useful metadata for LLM/AI workflows. Folding
it into the existing duration fetch keeps the tool fast and key-frugal.

## Output
Modified `src/main.rs`.
</objective>

<acceptance_criteria>

## AC-1: Title in default output
```gherkin
Given a valid YouTube URL and a configured YOUTUBE_API_KEY
When `yt <url>` runs with no field flags
Then the pretty-printed JSON output includes a "title" field holding the
     video's title alongside "transcript", "duration", and "comments"
```

## AC-2: Title-only flag
```gherkin
Given a valid YouTube URL and a configured YOUTUBE_API_KEY
When `yt --title <url>` runs
Then only the video title is printed to stdout (no JSON wrapper)
```

## AC-3: Single video-details request
```gherkin
Given the default path or the --duration/--title path
When video details are fetched
Then exactly one request is made to the youtube/v3/videos endpoint with
     part=contentDetails,snippet (duration and title come from one response,
     not two calls)
```

## AC-4: Missing-title resilience
```gherkin
Given a videos response whose item lacks a snippet.title
When video details are parsed
Then the tool does not panic and treats the title as an empty string
```

</acceptance_criteria>
